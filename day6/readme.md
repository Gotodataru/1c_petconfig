# День 6 — отчёт через СКД

## Что сделал

Реализовал пользовательский отчёт через Систему Компоновки Данных. Отчёт показывает продажи с группировкой по контрагентам и товарам, с фильтрацией по периоду. Вместо ручного программирования формы и табличного поля использовал СКД — это правильный подход в 1С для подобных задач.

Создал объект отчёта и подключил схему компоновки данных. Настроил набор данных на основе запроса к регистру накопления Продажи. Вручную указал ресурсы Количество и Сумма с агрегацией Сумма. Выстроил структуру вывода: группировка Контрагент, внутри неё Товар, дальше поля ресурсов. Добавил параметры НачПериода и КонПериода.

## Проблемы по порядку

Ресурсы не определялись автоматически. У числовых полей в таблице набора данных стоял флаг Группа, из-за этого они не появлялись в списке ресурсов. После снятия флага всё встало на место.

Для измерений в выпадающем списке предлагались только агрегатные функции. Решилось ручным вводом имени поля в ячейку Выражение.

При запуске отчёта падала ошибка про сравнение полей неограниченной длины. Оказалось, параметры периода были заданы в текстовом формате, плюс начало и конец периода были перепутаны местами. Установил тип Дата, очистил значения по умолчанию — ошибка ушла.

Отчёт формировал пустую структуру без данных. Через консоль запросов выяснил, что движений в регистре нет: тестовые документы были проведены ещё до финальной настройки регистра и кода проведения. Перепровёл все документы, данные появились.

## Итог

Отчёт работает стабильно. Данные из нескольких документов суммируются, фильтрация по периоду работает через стандартный диалог настроек. Структура вывода — стандартное дерево с возможностью сворачивать группы.
