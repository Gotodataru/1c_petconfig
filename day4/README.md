# День 4 — документ, регистр и проведение

## Что сделал

Создал три справочника: Товары, Контрагенты, Склады. Добавил документ ПродажаТовара с табличной частью — поля Товар, Количество, Цена, Сумма. Написал клиентский код, который автоматически пересчитывает сумму при изменении количества или цены. Создал регистр накопления Продажи с типом Обороты и написал модуль проведения — он записывает движения в регистр при проведении документа.

## Проблема с ВидДвижения

Пытался явно задать ВидДвижения = Приход в коде проведения — платформа выдавала ошибку «Поле объекта недоступно для записи». Оказалось, что для оборотного регистра с типом записи Регистратор это поле устанавливается автоматически, трогать его не нужно. Убрал из кода — всё заработало.
